Use event–division mappings when divisions perform different events — for example, Individuals do Events 1–4 but Teams only do Events 1–3.

**The rules:**

- **No mappings at all** — every event applies to every division. Most competitions never need this page.
- **Map an event to divisions** — only those divisions see it, score it, and get a leaderboard column for it.
- **Leave an event unmapped** — it stays visible to *all* divisions, even when other events are mapped. You only need to map the events that differ.
- **Multi-part events** — sub-events follow their parent's mapping. Map the parent only.

**To set up:**

1. Check the box for each division that performs each event.
2. Save.

**Where mappings apply:** the public workouts list, leaderboard columns, athlete score submission, and the division-variations section of the event editor all follow these mappings immediately.

**Before you publish results**, open Leaderboard Preview per division and confirm each division shows exactly the events it should.
