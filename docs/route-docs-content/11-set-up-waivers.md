title: How to set up waivers
type: markdown
routes:

---

1. Create a waiver with a title and the full legal text.
2. Set who must sign it:
   - **Required for athletes** — shown during registration; athletes (and teammates accepting invites) must sign before completing sign-up.
   - **Required for volunteers** — shown on the public volunteer signup form and when a directly-invited volunteer accepts.
   - A waiver can require both, or neither.
3. Reorder waivers to control the order athletes see them.

**Unchecking "required for athletes"** keeps the waiver manageable here but hides it from registration — useful for retiring a superseded version without losing its signatures.

**Catching missed signatures:** for in-person competitions, the day-of check-in kiosk shows each athlete's waiver status and lets them sign on the device at the door — so a teammate added late or a pre-signed-invite edge case never blocks check-in.

**Tracking:** signature status per athlete is on each registration's detail page, and volunteer signature columns are on the Volunteers roster. Signatures are timestamped legal records — they can't be edited or signed on someone's behalf.
